## ADDED Requirements

### Requirement: Bubbles are gated by proximity, cooldown, and probability
The system SHALL only fire a chat bubble when all three conditions are met: a player is within 8 blocks, the NPC's per-bubble cooldown has elapsed (45 seconds minimum), and a 15% probability roll succeeds.

#### Scenario: All conditions met
- **WHEN** a player is within 8 blocks, cooldown has elapsed, and 15% roll succeeds
- **THEN** a bubble fires and the NPC's cooldown resets to 45 seconds

#### Scenario: No player nearby
- **WHEN** no player is within 8 blocks of the NPC
- **THEN** no bubble fires regardless of cooldown or roll

#### Scenario: Cooldown not elapsed
- **WHEN** a player is within 8 blocks but cooldown has not elapsed
- **THEN** no bubble fires

### Requirement: Bubble content is generated by Claude API using task context and personality
The system SHALL call Claude API (claude-haiku-4-5) with the NPC's current task title and desk personality to generate a humorous one-liner.

#### Scenario: Successful API call
- **WHEN** a bubble fires and Claude API responds within 3 seconds
- **THEN** the API-generated one-liner is displayed as the NPC's chat bubble

#### Scenario: API timeout or failure
- **WHEN** a bubble fires but Claude API does not respond within 3 seconds or returns an error
- **THEN** the system selects a fallback phrase from the desk's pre-written personality phrase bank and displays it as the bubble

### Requirement: Bubble prompt uses personality modifier
The system SHALL inject the desk personality description into every Claude API bubble prompt.

#### Scenario: Prompt construction
- **WHEN** generating a bubble for any NPC
- **THEN** the prompt sent to Claude API is: "You are a [personality description] office worker. Paraphrase this task title as a casual one-liner you'd say at your desk: '[task title]'. Max 12 words. No quotes."

### Requirement: Each desk has a fallback phrase bank
The system SHALL maintain a minimum of 10 pre-written fallback phrases per desk personality for use when Claude API is unavailable.

#### Scenario: Fallback selection
- **WHEN** the fallback phrase bank is used
- **THEN** a phrase is selected randomly from the bank without repeating the most recently used phrase
